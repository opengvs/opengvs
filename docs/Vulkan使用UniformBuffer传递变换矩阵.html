椎体有四个顶点，需要绘制四个三角形，所以顶点数组中包含了12个顶点数据。下面我们分步骤描述Vulkan程序如何向Shader中传递UniformBufferObject数据：

创建描述符集布局对象VkDescriptorSetLayout 并在PipeLine中引用该对象

我们查看以前createGraphicsPipeline()函数原代码，找到：

//设置创建pipelineLayout对象时设置的相关信息，应用程序主要通过pipelineLayout的设置与Shader中的uniform变量交换信息 vk::PipelineLayoutCreateInfo pipelineLayoutInfo; pipelineLayoutInfo.setLayoutCount = 0;//我们这里使用的Shader没有需要传输的uniform变量 pipelineLayoutInfo.pSetLayouts = nullptr; pipelineLayoutInfo.pushConstantRangeCount = 0; // Optional pipelineLayoutInfo.pPushConstantRanges = 0; // Optional

//pushConstant是Vulkan中的一种特殊的uniform变量，它允许应用程序在渲染过程中动态地传递数据到着色器中 pipelineLayoutInfo.pushConstantRangeCount = 0;//设置Shader中需要应用程序传递的pushConstant变量的个数 vk::Result result = logicDevice.createPipelineLayout(&pipelineLayoutInfo , nullptr, & pipelineLayout); if (result != vk::Result::eSuccess) { throw std::runtime_error("failed to create pipeline layout!"); }

部分，我们可以看到创建pipelineLayoutInfo对象时，传入的VkPipelineLayoutCreateInfo信息即不包含Layout信息，也不包含ConstantRange信息，这是因为前面程序不需要有程序向Shader中传递顶点信息之外的数据。本例中我们需要向Shader中传递UniformBufferObject对象，所以需要向PipeLine中传递Layout对象，在Vulkan中是通过VkDescriptorSetLayout完成的，首先需要创建该对象：

创建descriptorPool对象和descriptorSet对象，将uniformBuffer对象数据传递给Shader的UniformBufferObject对象

到目前为止，我们设置了PipeLine中的pipelineLayout对象，完成了uniformBuffer对象的创建，并将数据传递给了uniformBufferMemory。然而实际数据现在并没有传递给Shader中的UniformBufferObject对象，Vulkan应用程序中这种工作是通过descriptorPool对象和descriptorSet对象完成的。

xxxxxxxxxx
2
1

2
VkDescriptorSet  descriptorSet = nullptr;

我们将为每个VkBuffer资源创建一个descriptorSet，以将其绑定到Uniform buffer描述符，Vulkan中描述符集不能直接创建，必须从描述符池中分配，定义新函数createDescriptorPool()。

VkDescriptorPoolSize描述描述符集将包含哪些描述符类型以及它们的数量（为每个帧都分配），

VkDescriptorPoolCreateInfo，除了可用的单个描述符的最大数量外，我们还需要指定可以分配的描述符集的最大数量。

描述符集分配通过VkDescriptorSetAllocateInfo，需要指定要分配的描述符池、要分配的描述符集的数量（为每个帧都分配）以及它们所基于的描述符布局vkAllocateDescriptorSets，不需要显式地清理描述符集，因为当描述符池被销毁时描述符集现在已经分配完毕，但是仍然需要配置其中的描述符。
