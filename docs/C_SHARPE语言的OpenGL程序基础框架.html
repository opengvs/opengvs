C#语言的OpenGL程序基础框架

刘文庆

C#语言开发OpenGL程序需要借助第三方提供OpenGL封装库，目前比较流行的C#封装库是OpenTK。OpenTK是一个开源的跨平台C#库，封装了OpenGL、OpenAL和OpenCL等底层图形、音频和计算API，主要用于3D图形编程、游戏开发和科学计算‌。它提供对多平台（Windows、Mac OS、Linux）的支持，并通过面向对象的设计简化了复杂图形编程的流程。‌‌为了追求更加贴近最新的技术前沿，我们的演示程序使用OpenTK 4.9.3开发包，该封装包支持.Net8.0版本，所以本文的演示历程可以移植到比较新的程序设计需求中。

由于OpenTK 4.8以后不再支持OpenGL兼容模式，要求用户必须使用OpenGL核心模式，所以本部分的几个OpenGL学习例程不再设计传统的OpenGL绘制方面的代码。

在OpenGL核心（core）模式下，离不开Shader(GLSL)与缓冲区的几个概念，作者C++版本的OpenGL程序设计部分详细讲解了VBO、EBO、VAO的几个概念，在本部分不做详细的概念验证，仅仅作一下概念介绍：

① Shader(GLSL)：在《三维图形学 openGL/Diret3D中变换原理》一文中详细介绍的OpenGL绘制图形过程的图形管线，GLSL就是OpenGL图形开发平台API 提供的在图形绘制过程中让用户自行设计部分GPU中运行的程序，一般分为顶点着色器（Vertex Shader）、片段着色器（Fragment Shader），有时会用到几何着色器（Geometry Shader）。如果不使用自定义的着色器程序，OpenGL会自动选择系统默认的着色器代码。为了简化程序代码，本文使用的着色器代码如下：

VAO（Vertex Array Object，顶点数组对象）：用于管理VBO和EBO的状态（如顶点属性指针、绑定的缓冲区等），简化渲染流程。一个VAO可记录多个VBO的配置，避免每次渲染时重复设置‌。

对于VAO、VBO、EBO队形，在OpenGL程序中均需要创建对象、传输数据、应用队形绘制图形、删除队形等工作过程，对于C#语言是面向托管.Net架构的，只需要创建new对象，不需要程序显示删除对象，而OpenGL对象是面向非托管应用的，需要显示删除他们，为此我们设计了如下的基础程序架构：

VAO、VBO、EBO、Shader类通过强制实现IBindable和IDisposable接口。实现了对象的非托管资源自动化删除过程，当然建议程序在退出时显示调用Dispose函数删除非托管的OpenGL对象。
