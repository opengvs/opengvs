作者：刘文庆

上一个程序主要是通过基本的OpenTK框架构建了一个基础的二维显示程序，在OpenGL窗口中显示一个三角形，并且通过界面操作完成二维图形的变换。本部分程序将带领同学分步骤的创建一个简单的OpenTK框架下的三维程序，并通过软件界面控制三维程序中相机的主要变换操作。

Camera类主要完成观察流水线中相机变换中视矩阵和透视投影变换的透视投影矩阵的获取，其输入参数分别是：

视变换：位置变量EyePosition、位置变量LookatCenter、向上的矢量Up；

透视投影变换：近平面距离Near、远平面距离Far、观察者水平张角Fov、观察椎体的宽高比值AspectRatio。

本文中用到的视矩阵计算过程可以参阅本博客中三维图形学中相机变换矩阵的推导过程文章内容

透视投影矩阵的计算过程可以参与本博客中三维图形学中的透视投影公式文章内容。

从上述代码我们可以看出，本程序主要定义了两个立方体对象，一个Shader对象，一个主相机对象m_MainCamera。为本程序使用的Shader源代码中由下面一行代码：

gl_Position = projection * view * model * vec4(vPosition, 1.0f);

表示在VretexShader中外部物理的顶点坐标会乘以世界坐标变换矩阵model、乘以相机变换矩阵view、乘以投影变换矩阵projection，最后得到规划化的投影坐标gl_Position全局变量，然后向下传给其他渲染管线如几何渲染管线和片源渲染管线。
