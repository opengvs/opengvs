Step-2-12 创建命令缓冲区池和命令缓冲区

Vulkan 系统设计程序过程中所需要执行的各种命令，比如绘制指令和内存传输指令并不是直接通过函数调用执行的。需要将所有要执行的操作记录在一个指令缓冲对象，然后提交给可以执行这些操作的队列才能执行。这使得我们可以在程序初始化时就准备好所有要指定的指令序列称为命令缓冲区CommandBuffer，在渲染时直接提交执行。这样就能够高效的使用多线程方式提交指令。我们只需要在需要指定执行的使用，将命令缓冲对象提交给Vulkan 处理接口。

创建命令缓冲前需要构建命令池CommandPool对象。命令池CommandPool对象用于管理命令缓冲CommandBuffer对象使用的内存，并负责命令缓冲CommandBuffer对象的分配。一个应用程序的指令缓存数量是非常多的，Vulkan API 设计师为了最大的提高性能，命令缓存区的分配通过 VkCommandPool 完成，从而降低多个指令缓存之间的资源创建带来的性能消耗。他们之间的关系如下图所示：

⑤ 创建命令缓冲区CommandBuffer对象

创建完成命令池CommandPool对象后，就可以创建命令缓冲区CommandBuffer对象，网上有个例子创建缓冲区CommandBuffer对象的数量与VkFrameBuffer对象的数量保持一致，其实并没有必要，真正的系统中，特别是在多线程渲染环境下可能创建N多个CommandBuffer对象，分别保存不同渲染需求的命令队列。作为入门等级的教程，本例我们只创建一个命令缓冲区CommandBuffer对象，在以后Vulkan多线程渲染章节，我们会进一步改造程序结构，添加任意多个命令缓冲区CommandBuffer对象。创建命令缓冲区CommandBuffer对象的代码如下：

命令缓冲区有两个级别：

一级命令缓冲区：对应VK_COMMAND_BUFFER_LEVEL_PRIMARY ，命令缓冲区可以被直接提交给硬件设备命令执行队列

二级命令缓冲区：二级命令缓冲区在开始录制时需指定继承信息（包含渲染通道在内的一些上下文参数），二级命令缓冲区必须从一级命令缓冲区执行。对于有多个子通道的渲染通道，通过在主线程中录制一级命令缓冲区，另一线程中将某个子通道中的命令录制在二级命令缓冲区中，可能可以在一定程度上缩减CPU一侧的执行时间。

⑥ 设计录制命令函数recordCommandBuffer

命令缓冲区CommandBuffer对象在程序绘制阶段可能处于多种状态：初始化状态、命令录制状态、可执行状态、待决状态、无效状态。这些状态之间的关系通过下图描述如下：

  • 如图所示，提交命令缓冲区之后、命令执行完之前的状态为待决状态（提一下以强调）。

  • 若命令缓冲区中涉及到的Vulkan对象被销毁，则命令缓冲区从录制或可执行状态进入无效状态。

  • 对于可反复录制（从指定了VK_COMMAND_POOL_CREATE_RESET_COMMAND_BUFFER_BIT的命令池中分配）且每次录制完只提交一次（开始录制时指定VK_COMMAND_BUFFER_USAGE_ONE_TIME_SUBMIT_BIT）的命令缓冲区，执行完以后可以手动将其重置，但通常不需要这么干，重新录制命令缓冲区时会将其自动重置到初始化状态（然后紧跟着进入录制状态）。
